# Contributing

Thank you for your interest in this project.

This repository is primarily intended as a **technical portfolio example** of an internal-style automation applied to operational, compliance and data workflows.

## Scope

Small improvements are welcome, such as:
- documentation refinements
- code organization
- robustness and readability improvements

Changes that require real Zendesk data, internal credentials or proprietary information will **not** be accepted.

## How to contribute

1. Fork this repository
2. Create a branch (`feat/your-feature` or `fix/your-fix`)
3. Keep changes small and focused
4. Open a Pull Request explaining:
   - what was changed
   - why it was changed
   - how it was tested (if applicable)

## Data and security

- Do **not** include real tickets, PDFs, IDs or credentials
- Use only synthetic or placeholder data if needed
- Respect data protection regulations and internal policies
